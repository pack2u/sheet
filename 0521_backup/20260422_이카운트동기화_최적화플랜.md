# 이카운트 동기화 최적화 플랜 (ecount.gs)

사용자님의 요청에 따라 기존 시스템 점검 내역 중 **이카운트 동기화(ecount.gs) 부분의 렌더링 지연(flush) 최적화**에만 집중하여 수정 계획을 수립했습니다.

## User Review Required

> [!TIP]
> 백그라운드(새벽) 배치 스케줄러가 동작할 때는 화면을 갱신해 줄 사용자가 없으므로, 불필요한 `SpreadsheetApp.flush()` 호출을 건너뛰도록(Bypass) 분기 처리합니다. 반면, 사용자가 모달 팝업에서 버튼을 눌러 동기화할 때는 기존처럼 실시간 렌더링이 되도록 유지합니다. 
> 이 방식을 통해 **새벽 자동 동기화 속도를 2~3배 이상 크게 향상**시킬 수 있습니다. 코드 변경안을 확인하시고 승인해주시면 바로 반영하겠습니다.

## Proposed Changes

### [MODIFY] ecount.gs

1. **파라미터 추가 (`isBackground`)**
   `ecountStep2`, `ecountStep3` 함수 정의 부분에 `isBackground` 파라미터를 추가하여, 백그라운드 실행 여부를 구분합니다.

2. **flush 우회 로직 적용**
   `SpreadsheetApp.flush()` 실행 코드를 `if (!isBackground)` 조건문으로 감싸서, 백그라운드일 때는 플러시하지 않고 모아서 처리하도록 변경합니다.

3. **백그라운드 함수 (`runDailyEcountBatch`) 수정**
   새벽 4시에 도는 백그라운드 작업에서 `ecountStep2`와 `ecountStep3`를 호출할 때 `isBackground` 값을 `true`로 넘겨주도록 수정합니다.

#### 주요 코드 변경 사항

**`ecountStep2` 수정 (약 39번 라인 및 139번 라인)**
```javascript
// 기존
// function ecountStep2(auth) {
// 변경
function ecountStep2(auth, isBackground) {
```
```javascript
// 기존 (139번 라인 근처)
//       SpreadsheetApp.flush();
// 변경
      if (!isBackground) SpreadsheetApp.flush();
```

**`ecountStep3` 수정 (약 147번 라인, 227번 라인, 265번 라인)**
```javascript

```
```javascript
// 기존 (227번 라인 근처)

```
```javascript

               if (!isBackground) SpreadsheetApp.flush();
```

**`runDailyEcountBatch` 수정 (약 415번 라인)**
```javascript

//     let auth = ecountStep1();
//     ecountStep2(auth);
//     ecountStep3(auth);
//     SpreadsheetApp.flush();
// 변경
    let auth = ecountStep1();
    ecountStep2(auth, true); // 백그라운드 모드 활성화
    ecountStep3(auth, true); // 백그라운드 모드 활성화
    SpreadsheetApp.flush();  // 여기서 한 번만 일괄 플러시
```
