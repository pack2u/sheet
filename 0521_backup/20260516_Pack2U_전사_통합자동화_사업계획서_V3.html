As-Is Analysis & Necessity

왜 지금 당장 '자동화'가 필요한가? (위기 진단)

물리적 한계: 3,500개 SKU의 압박과 인력 번아웃

현재 3,500가지의 막대한 제품군(SKU)을 관리 중이며, 단기 내 500여 가지의 신제품이 추가될 예정입니다.
이는 이미 인간의 뇌와 엑셀 수작업으로 감당할 수 있는 한계점(Threshold)을 완전히 돌파한 상태입니다.

❌ 수동 트래킹 불가 (단가/재고 연동 실패)
❌ 치명적 휴먼 에러 기하급수적 증가 (오배송/로스)
❌ 전화/카톡 폭주로 인한 CS 인력 극심한 과부하

물류: 모래성 같은 엑셀

VLOOKUP 의존. 열(Column) 1개만 어긋나도 4,000개 제품의 송장이 연쇄 붕괴. 매일 폭탄 돌리기식 수동 운영.

콘텐츠: 수작업 병목

신제품 500개 입고 시 썸네일 누끼부터 HTML, 전산 등록까지 전 과정을 인간이 수작업. 출시 지연의 주범.

세일즈: 수수료 종속

외부 마켓 의존도 90% 이상. 자체 데이터 확보 불가로 4,000개 상품에 대한 단골 재구매 유도(Push) 불가능.

1. Core Logistics Engine

코어 물류망 전면 개편 (엑셀 파괴)

"열(Column) 의존성 탈피 및 무결성 확보"

기존 시트의 수식(VLOOKUP) 의존도를 0%로 만듭니다. 모든 데이터는 JSON 객체(Key-Value 쌍) 형태로 변환되어 백엔드에서 처리되므로, 사원들이 실수로 시트의 행렬을 지워도 시스템은 붕괴하지 않습니다.

  • 데이터 모델링: 비정형 엑셀 데이터를 정규화된 RDBMS 스키마 형태로 파싱하여 메모리 적재
  • 무인 자동 매핑: 협력업체 상품명과 이카운트 품목코드(Item Code) 간의 Levenshtein 유사도 기반 자동 매핑 알고리즘 적용
  • 보안 격리: API 인증키 PropertiesService 완전 은닉 및 IP 기반 접근 통제(ACL)

이카운트 OAPI 다이렉트 브리지 및 큐(Queue) 아키텍처

엑셀 다운로드 후 수동 업로드 방식은 전면 폐기됩니다. "다이렉트 브리지(REST API)"를 통해 실시간 트랜잭션을 발생시킵니다.

  • API 규격: 이카운트 SaveOrder (주문서), SaveSale (판매전표) 엔드포인트 직접 호출
  • 세션 유지: 로그인 Session ID를 캐싱하여 불필요한 인증 요청 최소화 (Rate Limit 방어)
  • 장애 복원력 (Resilience): 통신 오류 발생 시 메시지를 데드레터 큐(DLQ)에 보관 후, 지수 백오프(Exponential Backoff, 1분➔2분➔4분) 알고리즘으로 자동 재시도

B2B Vendor Integration

협력업체 30개사 직결망: 발주/소통의 완전 자동화

현재의 난맥상: 파편화된 발주 채널

20~30여 개 협력업체의 발주 방식이 엑셀, 이카운트 개별 발주, 전화, 카카오톡 등으로 제각각 흩어져 있습니다.

  • 매일 아침 10가지가 넘는 채널의 발주 내역을 한 엑셀로 취합하는 데에만 2시간 소요
  • 단가 변동 시 일일이 전화나 카톡으로 고지해야 하는 소통 비용 발생
  • "재고 있나요?", "언제 출고되나요?" 등 반복적인 단순 확인 전화 폭주

해결책: Pack2U 파트너 전용 웹앱 (Direct Input)

모든 협력업체에게 B2B 전용 Web App 접속 계정을 발급합니다. 파트너사가 직접 발주를 입력하면 Pack2U 생태계로 즉시 다이렉트 연동됩니다.

  • 실시간 재고/단가 동기화: 파트너사는 앱에서 실시간 재고를 확인하고, 본인 업체에 설정된 등급별 단가로 즉시 발주
  • 취합 업무 Zero: 파트너사가 버튼을 누르는 순간 이카운트 ERP 전표 생성 및 WMS 피킹 리스트로 즉각 꽂힘
  • 정산(월마감) 자동화: 매월 말일, 파트너사별 발주 누적 데이터가 엑셀과 PDF 명세서로 자동 생성되어 카카오 알림톡으로 발송됨

"협력사와의 소통(전화/카톡)과 발주 취합(엑셀) 과정을 전면 제거하여, 서로간의 행정적 마찰을 없애고 오직 비즈니스 성장에만 집중할 수 있는 B2B 생태계를 구축합니다."

HR Strategy

인력의 재배치 (Workforce Reallocation)

단순 노동자에서 "가치 창출자"로

자동화를 통해 일자리를 줄이는 것이 목적이 아닙니다. 직원들을 소모적인 작업에서 해방시켜, 매출을 직접적으로 견인하는 부서로 전면 재배치합니다.

수동 송장 복사/붙여넣기 담당 (2명)
➡️ VIP 전담 어카운트 매니저 배치
상세페이지 포토샵 합성 담당 (1명)
➡️ 신제품 소싱 및 트렌드 기획자로 전향
단가/재고 엑셀 업로더 (1명)
➡️ 쇼핑앱 푸시 기획 및 프로모션 마케터

조직의 체질 개선

기계가 제일 잘하는 일(데이터 복사, 산술 계산)은 기계에게 100% 맡깁니다.

인간만이 할 수 있는 일(고객의 마음을 읽는 것, 불만 고객을 단골로 바꾸는 대화, 창의적인 상품 소싱)에 회사의 모든 인적 역량을 쏟아붓습니다.

CS Focus Strategy

압도적 CS 혁명과 단골 집중 케어

수동적 응대에서 "데이터 기반 능동적 케어(Proactive Care)"로

자동화로 확보된 매일 10시간 이상(월 200시간)의 잉여 리소스를 오직 '고객 감동(UX)' 창출에 투자합니다.

  • 선제적 알림 시스템: 택배사 API 연동으로 배송 지연 예상 시, 고객이 묻기 전에 선제적 사과 알림톡 및 보상 쿠폰 자동 발급
  • Fast-Track 환불/교환: 파손 사진 업로드 시, 묻지도 따지지도 않는 즉각 재발송 프로세스 도입 (LTV 극대화를 위한 의도적 로스 감수)
  • NPS(순수추천지수) 관리: 구매 3일 후 해피콜을 통한 만족도 조사 및 제품 피드백 수집

CRM 기반 VIP / B2B 고객 밀착 관리 체계

파레토 법칙에 기반하여, 매출의 80%를 책임지는 상위 20% 핵심 단골(프랜차이즈, 대형 식당)에게 자원을 집중합니다.

  • 전담 어카운트 매니저(AM) 제도: VIP 고객별 1:1 직통 라인 배정으로 대기 시간 없는 하이엔드 CS 제공
  • RFM 데이터 기반 영업: 대시보드의 RFM(Recency, Frequency, Monetary) 알고리즘이 고객의 재구매 주기를 예측. 재고가 떨어질 시점에 AM이 선제적으로 발주 유도(Outbound Call)
  • 폐쇄몰(Secret Mall) 혜택: 자체 쇼핑앱 내 B2B 전용 시크릿 권한을 부여하여 락인(Lock-in) 효과 극대화

AI CS Bot & Data Assets

8년의 노하우 집약: 자체 AI CS 봇 구축 및 데이터 자산화

왜 '데이터 자산화(Data Asset)'가 가장 중요한가?

지난 8년간 쇼핑몰을 운영하며 축적된 수만 건의 고객 문의(반품, 파손, 오배송, 단가 흥정) 로그는 돈으로 환산할 수 없는 기업 고유의 LLM 학습 데이터셋입니다.

  • 비정형 데이터 정제: 카카오톡 채널, 네이버 톡톡, 게시판의 대화 로그를 정형화된 JSON 형태로 파싱하여 Vector DB에 적재
  • 의도 분류(Intent Classification) 모델: 단순 배송 조회인지, 파손 클레임인지, 대량 구매 단가 흥정인지 AI가 즉각 분류
  • 지속적 학습(Continuous Learning): 매일 발생하는 신규 CS 데이터가 파이프라인을 타고 모델을 재학습(Fine-tuning)시켜 답변 정확도를 99%까지 끌어올림

Pack2U 전용 AI CS 봇 (RAG + Custom LLM)

수집된 8년치 CS 매뉴얼을 RAG(검색 증강 생성) 기술로 결합하여, Pack2U의 비즈니스 톤앤매너(Tone & Manner)를 완벽히 흉내 내는 AI 상담원을 배포합니다.

  • 시나리오 1 (새벽 발주): 새벽 3시 식당 사장님의 "A용기 5박스 단가 얼마야?" ➔ AI가 고객 등급 확인 후 도매가로 즉각 자동 답변 및 가주문 생성
  • 시나리오 2 (파손 클레임): "뚜껑이 깨졌어요 (사진 첨부)" ➔ AI Vision API가 사진 판독 ➔ 파손 확인 ➔ 사과 메시지와 함께 즉각 재발송 주문(ERP) 생성 및 10% 쿠폰 발급
  • 효과: 전체 CS 인바운드 콜의 90%를 AI가 방어(Deflect)하여 인간 상담원의 감정 노동을 원천 차단

Platform Strategy

현재: Google Sheets 환경의 한계

  • 데이터 한계 (Quota): 일일 API 호출 수 및 셀 갯수 제한(1,000만 개)으로 인해 대규모 스케일업 불가.
  • 퍼포먼스 저하: 수천 건의 송장을 한 번에 매핑할 때, 스크립트 실행 시간(Timeout 6분) 초과 위험.
  • 휴먼 에러 노출: 누구나 실수로 셀을 지우거나 필터를 잘못 걸면 전체 데이터가 꼬이는 원초적인 UI 문제.

미래: 전용 Web App 구축 시 장점

  • 무한한 확장성 (Scalability): 클라우드 데이터베이스(PostgreSQL 등)를 도입하여 1억 건의 주문도 지연 없이 0.1초 만에 검색/처리 가능.
  • 강력한 권한 제어 (Auth): 협력사나 직원이 볼 수 있는 화면과 메뉴를 DB 레벨에서 완벽히 통제 가능 (보안사고 Zero).
  • 모바일/PC 반응형 UI: 비좁은 엑셀 화면이 아닌 직관적이고 쾌적한 전용 대시보드 인터페이스 제공.
  • 실시간 양방향 연동 (WebSockets): 이카운트 ERP 업데이트 시, 웹 화면에서 즉각적으로 알림 팝업 및 데이터 자동 갱신.

"구글 시트는 훌륭한 초기 프로토타입(MVP) 도구였으나, 월 매출 수십억 단위의 엔터프라이즈 환경에서는 반드시 독립적인 Web App(클라우드 서버+DB)으로 이관되어야 합니다."

Risk Management

위험 관리 및 보안 대책

시스템 마비 (Downtime) 방어

구글 API나 이카운트 서버 점검으로 인한 일시적 통신 단절 시, 구축된 "메시지 큐(Queue)"가 모든 요청을 보관합니다. 서버가 복구되는 즉시 순차적으로 밀어넣어 데이터 증발을 100% 차단합니다.

협력사 권한 탈취 방어

협력업체 대리발송용 시트에는 최소한의 권한(Read/Write)만 부여하고 백엔드 소스코드는 숨깁니다. 임의의 엑셀 열 추가 조작 시도를 감지하면 스크립트가 실행을 즉각 거부하고 관리자에게 경고합니다.

안전한 전환(Rollback) 정책

빅뱅(Big-bang) 방식의 일괄 오픈이 아닌, 거래량이 적은 파트너부터 순차적으로 브리지를 엽니다. 에러 발생 시 버튼 한 번으로 구버전(엑셀 수동 방식)으로 즉시 회귀(Rollback)할 수 있는 '비상 스위치'를 대시보드에 탑재합니다.

Feasibility & Fallback Plan

이카운트 ERP 연동 타당성 검증 및 차선책 (Plan B)

Plan A: Ecount Open API (OAPI) 연동

현재 이카운트 ERP는 외부 시스템 연동을 위한 Open API(SaveSale, SaveOrder 등)를 제공하고 있어 기술적인 자동화 구현은 100% 가능합니다.

  • 구현 방식: JSON 페이로드를 통한 Session ID 인증 및 다이렉트 전표 생성
  • 잠재적 리스크: 이카운트 요금제에 따른 일일 API 호출 횟수 제한(Rate Limit), IP 화이트리스트(IP Whitelisting) 정책으로 인한 동적 IP(구글 서버) 차단 가능성, API 서버의 간헐적 불안정성

Plan B: RPA (Robotic Process Automation) 봇 도입

만약 API 호출 제한에 걸리거나 비용/정책상 API 사용이 불가능할 경우를 대비한 완벽한 차선책(플랜 B)이 준비되어 있습니다.

  • 자동화 엑셀 생성: Web App 또는 GAS가 인간을 대신해 완벽한 '이카운트 전용 양식 엑셀 파일'을 백그라운드에서 생성하여 특정 클라우드 폴더에 떨굽니다.
  • 무인 RPA 봇(Python/Selenium): 사무실 내 켜져 있는 미니 PC(또는 클라우드 가상머신)에 설치된 RPA 프로그램이 이카운트 웹사이트에 자동으로 로그인합니다.
  • UI 자동 업로드: RPA 봇이 '웹 자료올리기' 메뉴를 클릭하고 생성된 엑셀 파일을 업로드하는 과정을 인간의 손 없이 100% 모방(Simulate)합니다.

결론적으로, 이카운트 API 환경이 제한적이더라도 [서버리스 API (Plan A) ➔ 백엔드 무인 RPA 봇 (Plan B)]의 이중화 전략을 통해 인간의 개입이 필요한 구간은 완벽히 제거됩니다.

실시간 재고 연동 (Inventory Sync)

[이카운트 ERP ↔ Web App ↔ 사방넷]으로 이어지는 삼각 동기화 아키텍처를 구축합니다.

  • 물류 창고에서 상품 출고 시 이카운트 ERP에 반영되며, 해당 변동 사항이 사방넷을 통해 전 마켓 품절 처리로 직결
  • 단골용 B2B 몰과 B2C 외부 마켓 간의 재고 간섭 및 오버셀링(품절 취소 CS) 원천 차단

AI 상세페이지 전 채널 원클릭 배포 (Synergy)

기존에 구축된 [상세Html제작기]가 생성한 고품질 디자인 에셋을 사방넷의 '상품 일괄 등록' 기능과 연계합니다. 신제품 포장재가 입고되면 하루 만에 쿠팡, 스마트스토어 등 50여 개 채널에 동일한 규격의 프리미엄 디자인으로 동시 런칭되는 마케팅 폭발력을 확보합니다.
