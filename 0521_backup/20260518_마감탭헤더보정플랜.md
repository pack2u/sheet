# 전용발주 마감탭 헤더 불일치 수정 작업 보고

## 작업 일자
- 2026년 05월 18일

## 문제 원인 파악
- "뉴파츠(HR)" 업체처럼 전용양식 헤더 구조(32열)가 나중에 변경되었거나, 기존에 생성된 '전용발주 마감' 탭이 과거 양식을 유지하고 있을 때 발생하는 문제입니다.
- 기존 로직은 마감 탭이 "처음 생성"될 때만 헤더를 세팅하도록 되어있어, 한 번 생성된 마감 탭은 원본 전용양식이 변경되어도 헤더 레이아웃이 갱신되지 않았습니다.

## 수정 및 조치 사항
1. **이동 시스템(아카이브) 로직 개선 (`_partnerExclusiveArchive.gs`)**
   - 발주를 마감탭으로 이동시킬 때마다 **항상 최신 '전용양식'의 헤더와 레이아웃을 확인하고, 낡은 헤더(초과분)는 삭제하며 모자란 열은 추가하여 동기화**하도록 수정했습니다.
   - 앞으로는 전용양식이 변경되더라도 발주 마감 시 자동으로 완벽하게 맞춰서 백업됩니다.

2. **기존 탭 일괄 보정 기능 추가 (`partnerRepairExclusiveArchiveHeaders`)**
   - 어긋난 상태로 존재하는 기존 마감 탭들을 즉시 일괄 고칠 수 있는 수리 툴을 만들었습니다.
   - **AS / 진단 > 복구 도구 > 전용발주 마감탭 헤더 보정** 메뉴 항목으로 추가했습니다.

## 해결 확인 가이드
- 바로 문제를 해결하기 위해 시트 메뉴에서 **[🛠️ AS / 진단] > [🔧 복구 도구] > [전용발주 마감탭 헤더 보정]** 메뉴를 한 번 실행해주세요.
- 기존에 레이아웃이 어긋나 있던 "뉴파츠 전용발주마감 양식"을 포함한 모든 업체의 마감 탭이 정상적인 최신 양식으로 일괄 복구됩니다.
