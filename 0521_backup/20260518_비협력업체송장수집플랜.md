## 핵심 배경 — 왜 임시탭이 필요한가

`대리공급업체 발주` 탭은 **오전/오후 두 번 초기화(덮어쓰기)** 됩니다.
- 오전에 붙여넣기 → 오전 데이터만 있음
- 오후에 붙여넣기 → 오전 내용 사라지고 오후 데이터만 있음

따라서 이 탭에 고유ID나 Push여부를 기록해도 → 초기화 시 사라짐 → 오후 발주가 이미 발주된 것으로 오인.

**해결책: 임시탭(대리발송_임시기록)을 persistent 버퍼로 사용**

---

## 올바른 전체 흐름

```
오전 발주 담당자가 [대리공급업체 발주] 탭에 붙여넣기
    ↓
① 수집 단계 (partnerCollectToTempTab 또는 기존 Push 시작 시)
   [대리공급업체 발주] 탭 읽기
   → 임시탭(대리발송_임시기록)에 저장
     - 수취인전화+이름 기준 중복 체크 (이미 있으면 스킵)
     - 신규 행: 고유ID 발급 (예: 0518-NP-A3F2C1)
     - 발주완료 여부 = 빈칸 (아직 미발주)

② 발주 단계
   임시탭의 "발주완료 = 빈칸" 행을 읽어서
   → prefix별로 _PEP_VENDOR_DIRECT_MAP_ 열매핑 적용
   → 해당 업체 전용양식에 Push
     ★ 전용양식 B열(적요)에 임시탭의 고유ID 기록
   → 임시탭 해당 행에 "발주완료" 표시

오후 발주 담당자가 [대리공급업체 발주] 탭에 붙여넣기 (오전꺼 사라짐)
    ↓
① 수집 단계 재실행
   - 오전에 이미 임시탭에 있는 사람 → 스킵
   - 오후 신규 발주건 → 임시탭 추가 + 새 고유ID 발급

② 발주 단계 재실행
   - 오전 기발주 행 → 스킵 (발주완료 표시 있음)
   - 오후 신규 행 → 전용양식 Push + 고유ID 기록

업체들이 전용양식 A열에 송장번호 입력
    ↓
partnerFetchInvoices() — 송장 수집
   → 전용양식 A열(송장번호) + B열(고유ID) 수집
   → 협력업체_발주허브에서 고유ID로 매칭 ← ★ 이름/전화 아닌 고유ID 사용
   → 허브에 송장번호 기록
   → 허브 미매칭 송장 → 비협력업체_송장매칭 탭 (로젠 37열 양식)
```

## 구현 현황

### ✅ 기존 완성 — 수정 금지

- `_PEP_VENDOR_DIRECT_MAP_` — 업체별 전용양식 열매핑
- `_PEP_VENDOR_COL_OVERRIDES_` — 업체별 열 오버라이드
- `partnerFetchInvoices()` — 송장 수집 메인
- `_po_collectUnmatchedInvoicesToSeparateTab_()` — 전용양식→비협력업체_송장매칭

### ✅ 임시탭 헬퍼 — 기존 존재 (수정 금지)

- `_pep_ensureNonPartnerTempTab_()` — 임시탭 생성/확인
- `_pep_loadTempTabUids_()` — 중복방지 키 로드
- `_pep_appendToNonPartnerTempTab_()` — 임시탭 행 추가

### ⚠️ 이번 세션에서 변경된 것 (검증 필요)

**`_pep_appendToNonPartnerTempTab_` 전화열 수정**
- 변경 전: `row[8]` (모바일) 사용
- 변경 후: `row[7]` (전화) 우선, `row[8]` 폴백
- → 실데이터로 H열=전화 맞는지 확인 필요. 틀리면 되돌려야 함

**`_po_checkNonPartnerTempTabMatches_` 버그수정**
- `found.invoice` → `found[0].invRaw` (배열구조 반영)
- → 이건 실제 버그 수정이므로 유지해도 됨

**`_po_pushTempTabMatchedToNonPartnerSheet_` 신규 추가**
- 임시탭 K열(송장번호) 채워진 행 → 비협력업체_송장매칭
- → 이건 맞는 방향. 유지

### ❌ 아직 미구현 — 핵심 작업

1. **임시탭 L열(발주완료) 추가**
   - `_PEP_NON_PARTNER_TEMP_HEADERS_`에 "발주완료" 추가

2. **모든 행(협력+비협력)을 임시탭에 저장**
   - 현재: 비협력업체 행만 임시탭에 저장됨
   - 필요: 협력업체 행도 임시탭에 저장 후 L열로 Push 추적

3. **발주 소스를 임시탭으로 전환**
   - 현재: 대리공급업체 발주 → 전용양식 (직접)
   - 필요: 대리공급업체 발주 → 임시탭 → 전용양식
   - 임시탭 B열(적요)에 고유ID 기록 → 송장수집 시 ID 매칭

4. **송장 수집 시 고유ID 매칭 적용**
   - 현재: 이름+전화 4자리 기반 매칭
   - 필요: 전용양식 B열(고유ID) 기반 매칭 우선
