# 폐기송장 탭 — 송장번호 입력 시 자동 정보 조회 기능 추가

## 작업 일자
- 2026년 05월 18일

## 작업 내용
폐기송장 탭에서 A열(송장번호)을 입력하면, 협력업체_발주허브에서 해당 송장번호를 자동으로 검색하여 **판매처(발주업체), 품목명, 수량, 수취인** 정보를 자동으로 채워주는 기능을 추가했습니다.

## 상세 수정 사항

### 1. 폐기송장 탭 헤더 확장 (`_partnerOrders.gs`)
- **기존**: `[송장번호, 사유, 등록일시]` (3열)
- **변경**: `[송장번호, 판매처, 품목명, 수량, 수취인, 사유, 등록일시]` (7열)
- 기존 3열 탭이 있으면 '폐기송장 탭 열기' 메뉴 실행 시 자동으로 7열 구조로 보정됩니다.
- 기존 데이터(사유, 등록일시)는 F열, G열로 안전하게 이동됩니다.

### 2. onEdit 자동 조회 로직 추가
- `_po_onEditVoidInvoiceAutoFill_()` 함수를 신규 작성했습니다.
- A열에 송장번호 입력 즉시 현재 시트 내 '협력업체_발주허브' 탭의 N열(송장번호)을 검색합니다.
- 매칭 성공 시: B열(판매처), C열(품목명), D열(수량), E열(수취인)이 자동 채워집니다.
- 매칭 실패 시: B열에 "(미확인)" 표시가 나타납니다.
- G열(등록일시)도 자동으로 현재 시각이 기입됩니다.

### 3. menu.gs onEdit 연결
- `onEdit(e)` 함수에 `_po_onEditVoidInvoiceAutoFill_(e)` 호출을 추가했습니다.

## 데이터 소스
- **1순위**: 현재 시트 내부의 `협력업체_발주허브` 탭 (simple onEdit에서 작동)
- **2순위**: 외부 시트 `1KIBSmjpMVKLGoAkbrcKyTr4LOflszwS_xtMzmRuvYWs`의 허브 탭 (installable trigger 필요)

## 주의 사항
- 기존 폐기송장 적용 로직(`_po_loadVoidInvoiceSet_`)은 A열만 읽으므로, 이번 확장과 **충돌 없이** 정상 작동합니다.
- '폐기송장 탭 열기' 메뉴를 한 번 실행하시면 기존 3열 탭이 자동으로 7열로 보정됩니다.
