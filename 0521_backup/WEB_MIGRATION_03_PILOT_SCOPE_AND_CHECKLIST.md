# Pack2U 파일럿 업체 선정 및 병행 운영 체크리스트

## 1. 파일럿 목적
- 웹 전환 전, 실제 운영 데이터를 기준으로 동기화/충돌/업로드 안정성을 검증한다.
- 전사 전환 전에 운영 표준 절차(SOP)를 확정한다.

## 2. 파일럿 대상 선정 기준
- 권장 3개 업체:
  1. **일반형(표준 배포)**: 주문량 중간, 수정 빈도 보통
  2. **고빈도형(대량 주문)**: 주문량 높음, 피크 시간대 부하 검증용
  3. **소비자형(DC 배포)**: 5/8/10% 할인 로직 검증용

### 제외 기준
- CUST_CD 누락 업체
- 최근 2주 내 매핑 충돌/파일 접근 장애 다발 업체

## 3. 파일럿 기간
- 2주(영업일 10일) 권장
- 주차별 목적:
  - 1주차: 읽기 동기화 검증/불일치 감시
  - 2주차: 입력/상태 전환 일부 웹 우선 실험

## 4. 사전 준비 체크리스트 (D-3 ~ D-1)
- [ ] `업체등급단가매핑` CUST_CD 100% 입력
- [ ] 대상 업체 배포시트 메타(AA~AG) 최신화
- [ ] 월마감 탭 키(`AZ1`) 정상 여부 확인
- [ ] 이카운트 업로드 로그 샘플 3일치 확보
- [ ] 운영 담당자/백업 담당자 지정

## 5. 일일 운영 체크리스트 (파일럿 중)
- [ ] 오전 시작 전 diff 리포트 확인(전일 데이터)
- [ ] 발주 수집 건수(시트 vs 웹) 대조
- [ ] 송장 반영 지연 건 추적
- [ ] 업로드 실패코드 집계/재처리
- [ ] CUST_CD 변경 요청 반영 여부 확인
- [ ] 월마감 집계(전체/순액) 비교

## 6. KPI(합격 기준)
- 주문 건수 diff: 일별 ±0.5% 이하
- 금액 diff: 일별 ±0.5% 이하
- 업로드 성공률: 98% 이상
- 송장 반영 SLA: 30분 이내 95% 이상
- 충돌 건수: 일 3건 이하(모두 당일 해결)

## 7. 파일럿 운영 절차
1. 시트 우선 입력 유지
2. 웹 동기화 결과 대조
3. 특정 액션(상태변경/취소처리)만 웹 우선 수행
4. 결과를 시트 역반영 후 검증
5. 일일 회고(이슈/원인/조치) 기록

## 8. 장애 대응
- 심각도 S1: 업로드 중단/대량 누락
  - 즉시 웹 쓰기 중단 -> 시트 단독 운영 전환
- 심각도 S2: 일부 업체/일부 기능 오류
  - 해당 업체만 웹 쓰기 중단 + 브리지 재처리
- 심각도 S3: 표시/지연 이슈
  - 다음 배치에서 보정, 운영 공지

## 9. 파일럿 종료 Go 조건
- KPI 5개 모두 충족
- S1 장애 0건
- S2 장애 3건 이하, 재발 없음
- 운영팀 승인(업무 시간 증가 없음)

## 10. 산출물
- 파일럿 결과 보고서(건수/금액/오류/조치)
- 운영 SOP v1
- 전사 확대 일정표(업체군 순서)
