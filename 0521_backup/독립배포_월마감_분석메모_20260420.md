# 독립배포시트 월마감·운영 메모 (2026-04-20, 갱신)

## 1. 분석 대상
- 메뉴: `5단계 [수동 보관] 조건부 발주내역 정리 및 마감`
- 함수: `archivePastOrders()` (`orderSyncManager.gs`)
- 관련 메뉴 연결: `menu.gs` → `archivePastOrders`

---

## 2. 현재 구현 상태 (갱신)

### 2.1 월마감·아카이빙
- 독립배포 파일(`[독립 배포]`)의 `발주 및 송장조회` 데이터를 순회
- 과거일자 + 완료건(송장 존재/취소/품절/발송완료)을 월별 탭으로 이동
- 월별 탭명: `(YYYY년 M월) 발주 마감` 형태
- 월별 탭에 행 전체 저장(송장/정산단가 포함), 메인 탭에는 미완료·당일건만 유지
- 수동 실행 + 일일 트리거 실행 가능

### 2.2 이전에 부족했던 부분 → 반영 완료
| 항목 | 내용 |
|------|------|
| `취소` / `반품` 열 | 월마감 탭에 체크박스 열로 추가 |
| 요약 영역 | 상단에 합계·건수 등 요약(정산 규칙은 운영에서 확정한 대로 수식 반영) |
| 시트 보호 | 월마감 탭 보호, `취소`/`반품` 열만 편집 허용 |
| 기존 탭 보정 | `5-1` 메뉴로 이미 만든 월마감 탭 레이아웃 일괄 보정 |
| 당월 빈 탭 준비 | `5-2` 수동 전체 생성, `5-3`/`5-4`로 매월 1일 자동 생성 예약 |
| 탭 이름 변경 대응 | 숨김 메타 `AZ1`에 `ARCHIVE_MONTH:YYYY-MM` 저장 → 표시 이름과 무관하게 해당 월 탭 식별 |

### 2.3 코드 위치
- 메뉴: `menu.gs` (5단계·5-1~5-4 등)
- 본체: `orderSyncManager.gs` — `archivePastOrders`, 월마감 탭 레이아웃·보호·보정 함수들

## 3. 독립배포시트에서 「분리·배포」할 때 반영하면 좋은 점

상품 허브(중앙 시트)와 **업체별 독립 배포 파일**의 역할을 나누면, 나중에 웹으로 옮길 때도 같은 경계를 유지하기 쉽습니다.

### 3.1 중앙(허브)에서만 관리하는 것
- **거래처 코드 `CUST_CD`와 업체명의 정본**: `업체등급단가매핑` 시트. 코드가 이름보다 정확한 매칭 기준.
- 배포 시트 **파일 ID**와 업체의 대응 관계.
- 상품 마스터·단가 반영 후 **「배포 시트 순차 업데이트」**(메뉴 6-1 등): 한 번에 전부가 아니라 **순차·배치**로 부하 분산, 커서로 이어서 처리.

### 3.2 독립 배포 파일(업체용)에 두는 것
- 일별 발주·송장·해당 업체 단가가 반영된 뷰.
- 월마감 아카이브 탭(요약·취소/반품 체크는 여기서 운영 입력).
- 숨김 메타 셀(스키마 버전, 배포 타입, DC율, 공지 스크립트 설치 여부, 월키 등) — **사용자가 셀을 지우지 않도록** 안내하는 것이 좋음.

### 3.3 업체에 안내할 때 꼭 넣을 말 (배포 체크리스트 요약)
1. **코드 수정 위치**: 업체 코드는 배포 시트 안이 아니라 **중앙 매핑 시트**에서 바꾼 뒤, **업데이트 기능**으로 배포 시트에 반영.
2. **월마감 탭 이름**: 바꿔도 되지만, **월 식별은 `AZ1` 메타**로 잡도록 설계됨 — 그래도 불필요한 이름 변경은 줄이는 것이 운영에 유리.
3. **자동 실행 시간**: 배포 시트 일괄 업데이트는 **업무 방해를 피하기 위해 야간(예: 매일 22시)** 등으로 예약 가능 — 메뉴/트리거 설명과 맞춰 둘 것.
4. **수동 복구**: 배포 시트의 `onEdit` 공지 스크립트가 빠진 경우 `배포시트_자동완성코드_복사용.txt`로 복구 가능(백업 스니펫).

## 4. 이전 논의에서 나온 「이점」과의 연결 (요약)

| 이점 | 독립배포시트에서의 반영 |
|------|-------------------------|
| 식별자 일관성 | `CUST_CD` + 파일 ID 우선, 이름은 표시용 |
| 부하·한도 대응 | 업데이트 순차 처리·커서·락, 평균 처리 시간 기반 runLimit 조정 |
| 장애 추적 | `업데이트실행로그`(배포 업데이트·에러코드), `자동화실행로그`(월마감·당월탭), 메뉴 「📋 자동화 마지막 실행 상태」 |
| 월마감 안정성 | 탭 보호, 취소/반품 열, 요약, 메타 월키(`AZ1`) |
| 스페셜 DC | 5% / 8% / 10% 소비자용 시트, 메타에 DC율 기록 |
| 장기 웹 이관 | `WEB_MIGRATION_*.md` 문서와 메타/로그 표준(`WEB_MIGRATION_04_...`)과 정합 |

## 5. 정산 합계·운영 규칙 (확정 시 반영)

- 합계 기준: `확정단가 × 수량` 여부, 취소/반품 시 전액 제외 등은 **사업 규칙 확정 후** 수식·문구만 조정하면 됨.
