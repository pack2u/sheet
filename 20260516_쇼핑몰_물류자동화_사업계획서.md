# 쇼핑몰 물류 자동화 시스템 구축 사업계획서 (Pack2U)

## 1. 사업 개요 및 목표

### 1.1 사업 배경
현재 Pack2U는 다수의 오픈마켓(스마트스토어, 쿠팡, 11번가 등)과 이카운트 ERP를 연동하여 물류 및 주문을 처리하고 있습니다. 기존의 수동 엑셀 작업과 파편화된 스크립트 기반 운영에서 벗어나, 데이터 파이프라인의 안정성을 확보하고 전 구간을 100% 자동화하는 "스마트 관제 탑" 수준의 물류 자동화 시스템을 구축하고자 합니다.

### 1.2 사업 목표
1. **전 구간 완전 자동화**: 주문 수집부터 재고 할당, 송장 매핑, 출고, 이카운트 전표 처리까지 인간의 개입이 필요 없는 Zero-Touch 파이프라인 구축.
2. **에러 및 로그 관제 고도화**: 중앙 집중식 에러 로깅 및 모니터링 체계를 구축하여 장애 발생 시 즉각적인 대응(Slack/Telegram 알림) 가능.
3. **확장성 및 유연성 확보**: 향후 AI 사진 가공, 마진 계산기, 모바일 PDA 바코드 연동 등 신규 기능 확장을 수용할 수 있는 견고한 아키텍처 마련.
4. **보안 및 권한 강화**: API 인증 키 외부화, 운영자/실무자 접근 권한 분리를 통한 시스템 보안성 확보.

## 2. 현재 시스템 구조 및 한계점

### 2.1 현재 시스템 아키텍처
* **이카운트 동기화**: `ecount.gs`, `moveToEcount.gs`, `fetchItem.gs` (품목/재고 로딩, 상태 갱신)
* **단가 허브/배포**: `priceManager.gs` (허브 시트 및 배포 시트 단가/주문 양식 배포)
* **발주/송장 통합 허브**: `orderSyncManager.gs` (배포 시트 주문을 통합 발주 DB로 취합)
* **OCR 송장 매핑**: `송장시스템/autoTracking.gs` (V4 광역 안전 엔진 도입 중)
* **협력업체 시스템**: `_partnerMenu.gs`, `_partnerOrders.gs` 등 (협력업체 발주/송장 수집 및 정산)

### 2.2 주요 한계점 및 개선 필요성
1. **보안 취약점**: 하드코딩된 API 인증 정보와 스크립트 저장소 접근 권한 문제.
2. **함수 충돌 및 열 인덱스 의존도**: 전역 함수(`onOpen` 등) 충돌 위험, 시트 열 위치 변경에 따른 연쇄 장애 가능성 (P0/P1 리스크).
3. **단일 스크립트 과부하**: 특정 파일(`priceManager`, `orderSyncManager`)에 역할이 과밀되어 유지보수 어려움.
4. **모니터링 부재**: 시스템 장애, API 실패율, 재고 부족 등에 대한 가시성 부족.

## 3. 완전 자동화 시스템 구축 방안 (차기 마스터플랜)

### 3.1 1단계: 코어 시스템 안정화 및 보안 강화 (1~2개월)
* **비밀키 외부화 및 권한 제어**: API 키, COM_CODE 등을 `PropertiesService`로 이관하여 보안 강화. 사용자별(운영자/파트너) 접근 권한 정교화.
* **코드 리팩토링 및 모듈화**: 역할별 디렉토리 분리(`core/`, `ecount/`, `pricing/`, `order/`, `tracking/`), 함수 네이밍 충돌 해소.
* **표준 로그 시스템 도입**: `시스템_로그` 탭 생성, 함수별 실행 시간, 성공/실패 여부, 에러 코드(예: `UPLOAD_VALIDATION_FAIL`) 중앙화.

### 3.2 2단계: 핵심 파이프라인 지능화 (3~4개월)
* **예외 큐(Queue) 및 재처리 자동화**: 전송 실패, 매칭 실패 건을 자동 큐잉하고 원클릭 또는 백그라운드 재시도(지수 백오프) 구현.
* **상태 머신 표준화**: 주문 상태(접수대기, 발송완료, 취소 등)를 Enum 기반으로 통일.
* **직접 수정 (다이렉트 전송) 엔진**: 메인 시트에서 수정한 단가/재고를 엑셀 업로드 없이 즉시 이카운트 API로 꽂아넣는 기능 완성.

### 3.3 3단계: 스마트 관제 및 확장 기능 통합 (5~6개월)
* **AI 및 유틸리티 시스템 통합**:
    * **AI 사진 가공 시트**: 상세페이지용 이미지 자동 생성 및 Drive 자동 분류 시스템 통합 (Gemini 기반).
    * **오프라인 단가 마진 계산기**: 원가/배송비/포장비를 기반으로 목표 마진율(예: 30%)에 맞춘 자동 단가 산출 시스템 도입.
    * **카카오 챗봇 연동**: 협력업체 검색 발주, 단가 챗봇 등을 고도화하여 모바일 접근성 극대화.
* **현장 모바일 연동**: 스마트폰 카메라 기반 바코드 스캐너 웹앱 구현, 현장 재고 수정 및 전표 전송 지원.
* **중앙 관제 대시보드 (Web 기반)**: 전체 주문 흐름, 미매칭 송장, API 에러율, 재고 부족 경고 등을 한눈에 보는 모니터링 대시보드 구축. 심각한 에러 발생 시 Slack/Telegram 알림 전송.

## 4. 기대 효과

1. **운영 리소스 절감**: 수동 송장 매핑 및 엑셀 업로드 시간 90% 이상 단축.
2. **에러율 제로화**: 견고한 에러 핸들링 및 재처리 큐를 통해 데이터 누락 방지.
3. **확장성 기반 마련**: 웹 DB 및 하이브리드(Cloud Run/Functions) 연동을 고려한 데이터 모델(UUID 기반) 정립으로, 추후 SaaS 형태의 플랫폼으로 확장 가능.
4. **의사결정 속도 향상**: 실시간 대시보드와 스마트 알림을 통한 즉각적인 물류 이슈 대응.
