# 발주마감 이동 수정 작업 결과 보고

## 작업 일자
- 2026년 05월 18일

## 작업 내용 요약
'전용양식'에서 '전용마감탭'으로 발주건을 이동할 때, 당일(오늘) 발주건까지 같이 이동되는 문제를 해결하기 위해 필터링 로직을 수정했습니다.

## 상세 수정 사항
- **대상 파일**: `_partnerExclusiveArchive.gs`
- **수정 위치**: `_pea_core_` 함수 내 날짜 비교 로직
- **변경 내용**:
  - 기존: `if (rowDateNum > todayNum)` (오늘보다 미래의 날짜만 남김)
  - 변경: `if (rowDateNum >= todayNum)` (오늘 날짜 및 미래의 날짜를 남김)
- **효과**: 당일 발주된 주문건들(오늘 날짜)은 마감탭으로 이동하지 않고 계속 '전용양식'에 남게 됩니다. 어제까지의 발주 내역만 마감탭으로 깔끔하게 이동합니다.

## 기타 확인 사항
- 수정된 코드는 다른 로직(UID 초기화나 시트 레이아웃 등)과 충돌을 일으키지 않는 독립적인 날짜 비교 부분입니다.
- 즉시 적용되었으므로 다음 번 마감 이동 작업 시 오늘 날짜 데이터는 안전하게 유지될 것입니다.
