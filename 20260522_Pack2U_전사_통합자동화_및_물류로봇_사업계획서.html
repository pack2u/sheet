Executive Summary

핵심 요약 및 지능형 로봇 비전

현상 진단: 수작업 번아웃과 확장 병목

30개 대리공급 및 위탁 업체와의 파편화된 이메일/카톡 발주 취합, 수동 송장 매핑, 그리고 수작업 지게차 피킹에 전체 인력과 시간의 75%를 유실하고 있습니다.

  • 시간 손실: 매일 4.5시간 이상이 엑셀 업로드와 수기 데이터 매칭에 소모됨.
  • 물류 지연: 수기 발주서를 인쇄해 사람이 창고에서 제품을 찾아 카트에 싣는 구시대적 정체 발생.
  • 확장 불가능: 신규 제품이 500개 추가되면 현장 피킹 병목으로 당일 배송 붕괴 확정.

구축 방향: 무인화 연동 풀필먼트 (Zero-Touch)

주문 수집 즉시 이카운트 ERP 전표가 자동 생성되고, WCS(창고제어시스템)가 AGV(무인 지게차)와 AMR(이송 로봇)을 제어하여 패킹 대까지 100% 무인 피킹을 완료합니다.

  • 무인 피킹/적재: 무인 지게차가 파레트를 이송하고, AMR이 개별 박스 피킹을 무인 실행.
  • 송장 연동 코어: 업체별 발주 Push 시 생성된 고유 ID(UID)와 역수집 송장을 0.1초 만에 대조 완료.
  • 지능형 의사결정: RAG AI 에이전트가 CS 문의 방어 및 발주량 모니터링 수행.

As-Is Analysis

왜 물류로봇과 시스템 자동화가 절실한가? (위기 진단)

물리적 임계치 도달: 3,500개 SKU 및 공간/인력 한계

현 취급 SKU는 3,500가지에 육박하며 매월 신상품이 밀려오고 있습니다. 수기 랙 매핑 및 지게차 기사의 숙련도에만 전적으로 의존하는 적재 방식은 이미 휴먼 에러율이 임계치를 넘어섰습니다.

✖ 품절/위치 정보 오류로 지게차 기사의 랙 탐색 지연
✖ 피킹 카트 혼선으로 인한 묶음 배송 누락
✖ 정산 단가 수동 기입 오류로 역마진 리스크 노출

사무운영: 유리벽 엑셀

셀이 하나 밀리거나 누락되면 수백 건의 주문서 UID가 뒤엉켜 송장 역수집 및 ERP 매칭이 완전히 차단됩니다.

창고운영: 인력 종속

지게차 숙련 기사가 부재할 시 적재 위치 파악이 차단되어 하루 출고량이 반토막 나는 심각한 인력 의존성 발생.

정산/단가: 사후 대조

위탁/대리 대금 및 할인율 정산 처리가 월말 엑셀 취합본에 의존하여 누락분 대사에 긴 시간이 낭비됩니다.

Platform Strategy

구글 시트 의존성 탈피와 Web App & WCS 기반 고도화

현행 시트/Apps Script의 물리 한계

  • 실행 제한 (Timeout): 구글 앱스 스크립트의 6분 타임아웃 제한으로 대량 송장 일괄 매칭 시 실패율 상승.
  • WCS(로봇제어) 연동 불가: 구글 시트 내부 환경에서는 물류 창고 로봇 제어 서버와의 양방향 초고속 소켓(Socket) 통신이 제한적임.
  • 화면 복잡도: 수천 줄의 행이 존재하는 시트를 작업자가 스마트폰 모바일 화면에서 제어하기 불가능.

자사 Web App + WCS 구축의 가치

  • AMR/AGV 실시간 관제: WCS 서버와 로봇 제어(Fleet Management System)가 밀리초(ms) 단위로 작업 동기화.
  • 보안 역할 분담: 관리자, 현장 작업자, 대리공급 협력사에게 권한을 완벽히 차단 부여(정보유출 원천차단).
  • 이카운트 API 비동기 큐: 초당 수십 건의 이카운트 ERP 저장 요청을 백그라운드 큐가 보장 처리.
  • 모바일 최적화: 모바일 반응형 페이지로 현장 작업자 스마트 스캐너와 완벽 연동.

07
Core Backend Spec

코어 백엔드 기술 사양 및 이카운트 OAPI 통합

스프레드시트 1단계 개선 성공 모델 반영

구글 스프레드시트 기반 MVP에서 확인된 데이터 무결성 보장 모델을 데이터베이스 백엔드로 정형화하여 이식합니다.

  • 중복 처리 완전 방어: 이카운트 판매 전표 기입 완료 시 해당 거래건 P열에 이카운트 업 완료 플래그를 마킹하여 동일 데이터의 이중 갱신 및 매출 꼬임 현상을 원천 방어합니다.
  • 트랜잭션 락킹 (Script Lock): 대리 Push 및 아카이브 등 다중 메뉴 호출 시 분당 충돌 방지를 위한 스크립트 락 메커니즘을 백엔드 트랜잭션 격리(Isolation Level) 레벨로 승격 구현합니다.

이카운트 API 비동기 브리지

안정적인 ERP 연동을 위해 REST API를 비동기 큐 방식으로 운영합니다.

  • 세션 캐싱 모델: 이카운트 로그인 세션 만료 시간을 캐시 DB에 추적 저장해 인증용 호출 횟수를 감축하고 속도를 3배 개선합니다.
  • 재시도 큐(Retry Queue): 이카운트 서버 점검 등으로 통신 단절 시 데이터 소실 없이 지수 백오프 로직을 적용해 자동 재입력합니다.

Sabangnet Integration

사방넷(Sabangnet) 3자 통합 연동 및 삼각 재고 동기화

다채널 주문 자동 수집 및 정형화

사방넷 오픈 API와 백엔드 수집 스케줄러를 직결하여 50여 개 오픈 마켓의 주문을 완전 자동 수집합니다.

  • 주문 포맷 단일화: 각 몰별로 상이한 주문자/수취인 양식을 단일 DB 규격으로 정밀 자동 파싱합니다.
  • 실시간 운송장 역전송: 물류 로봇 포장 존에서 스캔 인쇄된 송장번호를 사방넷 API를 통해 매칭 주문건으로 즉시 리턴하여 강제 전송시킵니다.

오버셀링 방지 삼각 재고 동기화

[이카운트 ERP(물리 실재고) ↔ 백엔드 DB ↔ 사방넷 쇼핑몰 재고]를 초단 단위로 동기화합니다.

  • 창고 내 로봇 출고 완료 정보가 감지되면 즉시 이카운트 전표 등록과 함께 사방넷 재고 수량을 차감 업데이트합니다.
  • 안전재고 미달 품목 발생 시 B2B 전용 시트 및 자사몰에서 즉각 품절 상태 처리하여 품절 취소 CS 리스크를 사전에 예방합니다.

B2B Vendor Integration

대리공급 및 위탁 협력업체 30개사 발주 연동 메커니즘

현행 대리공급 및 위탁 발주 프로세스 반영

중앙 상품정보 시트와 각 협력업체 배포 시트 간에 구현된 데이터 통신 스펙을 그대로 유지/강화합니다.

  • 이카운트 코드 전용 Push: 수집된 발주 정보 중 이카운트 품목코드 앞 2자리(Prefix)를 감지하여 협력업체 개별 스프레드시트의 전용양식 탭으로 주문을 Push합니다.
  • 복합 고유키 대조 (AX열): 중복 Push를 물리적으로 차단하기 위해 사방넷 주문 UID와 매핑 코드를 결합한 복합 고유키(UID|코드)를 생성하고 검증을 수행합니다.

역수집 자동화 및 상태 갱신 메커니즘

대리공급업체가 운송장 번호를 전용양식 A열에 채우면 역수집 모듈이 즉시 가동됩니다.

  • 대리공급_임시기록 상태 추적: Push 즉시 임시기록 Y열에 발주완료를 마킹하며, 협력사 송장 기입 감지 즉시 송장을 긁어와 X열에 기록하고 진행상태를 송장수집으로 전환시킵니다.
  • 연락처 앞자리 '0' 보호: 엑셀 기입 과정에서 수취인 번호의 맨 앞자리 0이 소실되어 생기는 오배송을 예방하기 위해, 해당 열 포맷을 텍스트형(@)으로 강제 유지하는 전처리 기술을 적용합니다.

Robot Logistics (NEW)

무인 지게차(AGV) 및 이송 로봇(AMR) 기반 무인 물류 자동화

창고 무인화 혁신: 로봇 군집(Fleet) 기반의 무인 피킹 센터

Pack2U 물류창고에 WCS(창고제어)와 동기화되는 AGV와 AMR 로봇을 도입하여, 사람이 창고 내를 도보로 이동하며 피킹하던 전통 방식을 로봇이 물건을 집어 사람 앞으로 가져오는 GTP (Goods-To-Person) 방식으로 전면 개편합니다.

1. 무인 지게차 AGV (Automated Guided Vehicle)

역할: 고중량 파레트 하역 및 하이랙(High Rack) 입출고 자동화

  • 지정된 주행 경로(QR 그리드 또는 라이다 슬램)를 따라 최대 1.5톤의 파레트를 입고 랙 6단 높이까지 자율 적재합니다.
  • WMS와 연동되어 품목별 출고 빈도에 따라 로봇이 자율적으로 회전율이 높은 자리에 파레트를 전면 배치합니다.

2. 자율주행 이송 로봇 AMR (Autonomous Mobile Robot)

역할: 다품종 소량 피킹 박스 및 파우치 포장 존 고속 이송

  • WCS로부터 주문 정보를 수신한 AMR이 해당 상품이 적재된 선반(랙) 하부로 진입해 선반 통째를 들어 포장 작업대 앞으로 이송합니다.
  • 장애물 감지 및 우회 기능이 내장되어 작업자와 충돌하지 않고 24시간 끊임없이 피킹 이송을 처리합니다.

Smart WMS & Robot Control

스마트폰 QR 연동 WMS와 로봇 제어(WCS/Fleet) 솔루션

스마트폰 Web 기반 입출고 검수 (WMS)

고가의 전용 PDA 단말기 대신 직원의 스마트폰 카메라로 QR 코드를 스캔해 즉각 처리합니다.

  • 스캔 즉시 ERP 등록: 제품 박스의 바코드/QR 스캔 시 실시간 REST API를 통해 이카운트 재고 조정 및 판매 기입이 완료됩니다.
  • 휴먼 검수 에러 제로: 주문서에 없는 오배송 품목 스캔 시, 스마트폰 화면에 강한 적색 경고와 진동이 울려 출고 에러를 원천 차단합니다.

WCS & Fleet Management 로봇 관제

수십 대의 AMR과 AGV가 좁은 물류센터 안에서 엉키지 않도록 지능형 제어 인프라를 구축합니다.

  • 지능형 경로 최적화: 복수 품목 합배송 시, 최단 거리 상에 있는 AMR들이 순차 배차되도록 알고리즘 제어합니다.
  • 실시간 배터리 관리: 로봇의 배터리 잔량을 수시 감지하여 출고 작업이 없는 유휴 시간에 자동 충전 스테이션으로 도킹 충전 명령을 내립니다.

Risk Management

시스템 안전 예외 방어 및 로봇 안전 조치

1. 로봇 오작동 및 안전사고 방어 (WCS LiDAR)

자율주행 AMR과 무인 지게차 AGV 도입에 따른 현장 충돌 리스크를 완전 차단합니다.

  • LiDAR 기반 이중 세이프티 센서: 전방 2.5m 내 작업자 감지 시 1단계 감속, 1m 진입 시 즉각 비상 강제 정지 처리합니다.
  • 물리적 비상 정지(E-Stop) 버튼: 현장 랙 요소요소 및 로봇 바디 상단에 직관적인 하드웨어 비상 정지 스위치 배치.

2. 트랜잭션 롤백 및 데드레터 큐 (DLQ)

구글 클라우드 및 이카운트 서버 마비 등 비상 전산 정지 상황을 방어합니다.

  • 비상 롤백 스위치: API 연동 에러가 다량 발생하여 시스템이 임시 중단될 시, 기존 구글 시트 기반 수동 작업 탭으로 연동 방식을 즉시 스위칭할 수 있는 롤백 시나리오 확보.
  • 데드레터 큐 보관: 실패한 Push/Fetch 발주 트랜잭션은 삭제하지 않고 전용 큐에 적재하여, 서버 복구 시 누락 없이 100% 일괄 처리합니다.
