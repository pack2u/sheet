## Pack2U 시스템 전체 정리 (2026-04-20)

### 1) 시스템 목적

- 독립배포 시트에서 발주를 수집해 `통합 발주 DB`로 모은다.
- 수집된 발주를 이카운트 판매전표로 업로드한다.
- 업로드 성공 건만 분리시트/정산으로 연결해 회계/정산 일관성을 유지한다.

---

### 2) 핵심 흐름 (운영 기준)

1. 독립배포 시트 생성/갱신 (`priceManager.gs`)
2. 배포 시트 발주 수집 → `통합 발주 DB` (`orderSyncManager.gs`)
3. 업체/CUST_CD 매핑 점검 (`vendorCustCodeManager.gs`)
4. 통합 발주 기반 이카운트 판매 업로드 (`salesUploadFromIntegratedOrders.gs`)
5. 업로드 성공 건 분리 전송 (`자체 시트분리시트`)
6. 월 정산 탭 생성/보관 (`독립배포_정산_YYYYMM`)

---

### 3) 주요 파일 역할

- `menu.gs`
  - 전체 메뉴 진입점
  - 실행 순서 기준 메뉴 정렬 완료

- `priceManager.gs`
  - 허브 생성/동기화
  - 독립배포 시트 생성
  - 신규 배포 생성 시 메타 저장:
    - `AA1`: 발주업체명
    - `AB1`: 거래처코드(CUST_CD)
    - `AC1`: 배포시트ID
  - 신규 배포 생성 직후 매핑 시트 자동 등록 호출

- `orderSyncManager.gs`
  - 독립배포 발주 수집 (`pullOrdersFromVendors`)
  - 공급사 전송/송장 회수/송장 분배
  - 과거 건 마감 아카이브

- `salesUploadFromIntegratedOrders.gs`
  - 통합 판매 업로드 오케스트레이션
  - 원천: `통합 발주 DB` + `판매현황` (우선순위 병합)
  - `orderLineKey` 중복 제외
  - 판매 업로드 로그/오류 로그 기록
  - 성공 건 확정분리 전송
  - 월별 정산 탭 생성/보관
  - 업로드 전 `CUST_CD` 미입력 업체 존재 시 강제 차단

- `vendorCustCodeManager.gs`
  - `업체등급단가매핑` 시트 자동 생성/갱신
  - 기존 배포시트 일괄 `CUST_CD` 반영
  - `CUST_CD` 미입력 행 빨간색 경고 표시

---

### 4) 현재 메뉴 순서(발주/업로드/정산)

- 1~5단계: 발주 수집/송장 동기화/마감
- 6단계: 업체명 자동수집/거래처코드 시트 갱신
- 6-1단계: 기존 배포시트에 거래처코드 반영
- 7단계: 통합발주 기반 판매 업로드(수동)
- 7-1단계: 오늘분만 판매 업로드(수동)
- 7-2단계: 판매업로드 자동 실행(10시/14시) 켜기
- 7-3단계: 판매업로드 자동 실행 끄기
- 8단계: 업로드성공 건 분리시트 전송
- 9단계: 당월 정산 탭 생성/갱신
- 9-1단계: 당월 정산 탭 보관본 생성

---

### 5) 데이터 기준 원칙

- 단가/상태 기준은 `통합허브시트`를 사용한다.
- 그룹은 초기 배포 생성 시점에 확정한다.
- 업로드/정산은 스냅샷 단가 기준으로 처리한다.
- 이카운트 업로드는 `CUST_CD` 매핑 완료가 선행 조건이다.

---

### 6) 운영 체크리스트

#### 초기/변경 시

1. 허브 동기화 정상 확인
2. `6단계` 매핑 시트 갱신
3. `CUST_CD` 누락(빨간 셀) 해소
4. `6-1단계` 기존 배포시트 반영

#### 일상 운영

1. 발주 수집 실행
2. 판매 업로드 실행(수동/자동)
3. `8단계` 성공 건 분리 전송
4. 오류 로그 확인 후 재처리

#### 월 마감

1. `9단계` 정산 탭 생성/갱신
2. 대사 확인
3. `9-1단계` 보관본 생성

---

### 7) 현재 보안/운영 정책

- 이카운트 인증정보는 현행 운영 정책에 따라 코드 내 유지(사용자 요청사항).
- 대신 업로드 전 검증/로그/중복제외/확정분리로 운영 안정성을 보강.
