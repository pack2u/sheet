# 🧠 파트 2. 고유 아이디(UID) 생존 분석

현재 가장 중요한 과제는 **W1(독립 배포판)**에서 생성된 [고유 아이디]가 **S_MAIN(판매현황)**을 거쳐 **S_SET(세트분류)** 등 여러 탭으로 쪼개질 때 **"과연 버려지지 살아서 송장 허브로 넘어가는가?"**입니다.

### ❓ 사장님께 드리는 검증 포인트

1. **엑셀 함수(`QUERY` 등)의 연결 고리**
   - [판매현황] 시트에는 네이버나 쿠팡에서 부여한 **원본 주문번호(스마트스토어 주문번호 등)** 칸이 존재하나요?
   - 만약 존재한다면, [판매현황]에서 [세트분류]나 [대리공급업체 분류] 탭으로 데이터를 자동으로 쪼개서 넘길 때, 그 **주문번호 칸도 함께 넘겨주도록 엑셀 함수가 구동되고 있나요?**

2. **만약 기존 함수가 주문번호 칸을 넘겨주지 않고 있다면?**
   - 기존 시트를 수정해야 합니다! 하지만 스크립트 대공사가 아니라, 기존 [세트분류], [대리공급업체 분류] 탭에 걸려있는 엑셀 수식(`=QUERY(판매현황!A:Z, "SELECT A, C, E...")`)에서 **고유아이디가 들어갈 기둥 알파벳 하나만 추가(`SELECT A, C, E, Z...`)**해주면 간단히 뚫립니다.

3. **역할 분담**
   - 기존의 시스템은 철저하게 **엑셀 수식(함수)**에 기반한 완전 자동화 구조네요. (최고입니다!)
   - 그렇다면 이 상황에서 저희의 플랜 B(스텔스 아이디 꼬리표 달기)를 적용하려면, 제가 로봇(스크립트)을 짤 영역은 독립 배포판 쪽과 제일 앞단(판매현황)뿐이고, **중간 배관(세트분류 등으로 물려주는 역할)은 사장님께서 기존 엑셀 함수를 아주 살짝만(아이디 열 하나 추가) 고쳐주시면** 파이프라인이 거대한 하나의 생태계로 완벽하게 연결됩니다.
