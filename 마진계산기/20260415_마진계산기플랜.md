# 오프라인 판매가(X열) 마진 계산기 구현 계획

사내 물류 시스템의 **'오프라인 판매가'**를 특정 마진율(기본 30%)에 맞춰 원클릭으로 계산 및 반영해 주는 전용 도구를 구축합니다.

## User Review Required
> [!IMPORTANT]
> - 신규 탭 이름은 **'단가마진계산기'**로 생성됩니다.
> - X열(오프라인 판매가) 계산 기본 공식: `ROUNDUP( ((N*V) + O + S) / (1 - 마진율), -3 )` (1,000원 단위 올림)
> - 계산 로직에 사용되는 변수: 
>   - N열: 박스입수량
>   - V열: 상품원가 (U*1.1 등으로 이미 계산된 값)
>   - O열: 박스배송비
>   - S열: 포장비
> 이 공식과 참조하는 열이 맞는지 확인해 주세요.

## Proposed Changes

### 1. 신규 탭(UI) 및 모듈 생성
#### [NEW] marginCalculator.gs
새로운 기능은 기존 시스템과 충돌을 막기 위해 완전히 독립된 파일(`marginCalculator.gs`)로 작성합니다.

- `setupMarginCalculatorTab()` 함수
  - **'단가마진계산기'** 탭을 자동 생성합니다.
  - A1:B1 에 "목표 마진율" 입력칸을 만듭니다. (기본값 `30%`, 사용자가 언제든 `20%`, `40%` 등으로 자유롭게 수정 가능)
  - 3행에 헤더를 작성합니다: `[이카운트 코드, 상품명, N/V/O/S 스냅샷, 기존 판매가(X), 신규 판매가(X), 계산 및 반영 결과]`

- `executeMarginCalculation()` 함수
  - 탭의 A열에 입력된 **이카운트 코드**들을 줄줄이 읽어 들입니다.
  - B1 셀에 적힌 **마진율**(예: 0.3)을 가져옵니다.
  - **'상품정보'** 탭(메인 DB)을 뒤져서 코드가 일치하는 상품을 찾습니다.
  - 해당 상품의 `N, V, O, S` 열 데이터를 가져와 `((N*V)+O+S) / (1-마진율)` 계산 후 1,000원 단위로 올림(ROUNDUP -3)합니다.
  - 상품정보 탭의 **X열**에 계산된 값을 강제로 즉시 덮어씌웁니다!
  - 작업 결과를 계산기 탭에 "✅ 반영 완료" 상태와 함께, 원 가격과 새 가격을 나란히 출력하여 비교할 수 있게 합니다.
