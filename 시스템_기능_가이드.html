📅 매일 오전 확인 루틴

이카운트-자동연동로그 확인 → 오류 있으면 이카운트 연동 진단 → 정상이면 발주 처리 시작

📦 발주 처리 순서

① 발주 수집 → ② 판매현황 갱신 → ③ 대리공급 Push → 공급사 작업 → ④ 송장 수집 → ⑤ 폐기송장 적용 → ⑥ 송장 배포

🗓️ 월 마감 순서

이카운트 매출 대사 → 월별 정산 Dry-run 확인 → 월별 정산 이동 → 전용양식 마감탭 이동 → 허브 아카이브

⚠️ 장애 대응 순서

Push 시스템 통합 진단 → 이카운트 연동 진단 → 거래처코드 매핑 진단 → 동기화 락 해제 (필요 시) → 재실행
⚠️ 절대 하지 말 것:
· 시트 열(Column)을 임의로 추가/삭제하지 않는다
· 파일명/시트명으로 업체를 식별하지 않는다 (설정 탭 B5/B6만 기준)
· 판매현황 복붙 전 사전 점검 없이 대량 반영하지 않는다
· 매핑 실패를 문자열 트릭으로 우회하지 않는다
✅ SSOT(정답 소스) 원칙:
· 거래처명/거래처코드: 각 협력업체 스프레드시트의 설정 탭 B5(거래처명) / B6(CUST_CD)
· 품목 정보: 이카운트 ERP (매일 자동 동기화)
· 단가: 허브 스프레드시트의 전체 그룹 단가표
